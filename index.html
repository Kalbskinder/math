Virustest bei Schafen

Die Falsch-Positiv-Falle verstehen

📋 Ausgangslage

In einer Schafherde ist 1 von 250 Schafen mit einem Virus infiziert.

Um das Virus zu erkennen, wird bei jedem Schaf ein Schnelltest durchgeführt. Der Test ist aber nicht perfekt:
🦠Ein infiziertes Schaf testet in 85% der Fälle positiv (Sensitivität)
✅Ein gesundes Schaf testet in 5% der Fälle fälschlicherweise positiv (Falsch-Positiv-Rate)

🌳 Schritt 1 — Das Baumdiagramm aufbauen

Zuerst teilen wir alle Schafe in zwei Gruppen: infiziert und gesund. Dann schauen wir, wie der Test bei jeder Gruppe ausfällt.
Stufe 1: Infiziert oder gesund? 
Start
 ├─ Infiziert  (1/250 = 0.004)
 └─ Gesund     (249/250 = 0.996)
Von 250 Schafen ist im Schnitt 1 infiziert und 249 sind gesund.
Stufe 2: Wie testet jede Gruppe? 
Start
 ├─ Infiziert (0.004)
 │    ├─ Test + (0.85)  → 0.004 × 0.85 = 0.0034
 │    └─ Test − (0.15)  → 0.004 × 0.15 = 0.0006
 └─ Gesund (0.996)
      ├─ Test + (0.05)  → 0.996 × 0.05 = 0.0498
      └─ Test − (0.95)  → 0.996 × 0.95 = 0.9462
Kontrolle (Astregel): 0.0034 + 0.0006 + 0.0498 + 0.9462 = 1.0000 ✓

💡 Schritt 3 — Das überraschende Ergebnis

Jetzt kommt die entscheidende Frage: Wenn ein Schaf positiv testet, wie wahrscheinlich ist es WIRKLICH infiziert?

🤯 Die Falsch-Positiv-Falle

Stellen wir uns 10'000 Schafe vor und rechnen mit konkreten Zahlen:

  • 🦠 40 Schafe sind infiziert (10'000 ÷ 250)
  • ✅ 9'960 Schafe sind gesund

Wie testet jede Gruppe?

  • ● Von 40 infizierten: 34 testen positiv (85% von 40)
  • ● Von 40 infizierten: 6 testen negativ (15% von 40)
  • ● Von 9'960 gesunden: 498 testen positiv! (5% von 9'960)
  • ● Von 9'960 gesunden: 9'462 testen negativ

Insgesamt testen 34 + 498 = 532 Schafe positiv.
Aber nur 34 von diesen 532 sind wirklich krank!

→ p(infiziert | positiv) = 34 / 532 ≈ 6.4%

Das heisst: Über 93% der positiv getesteten Schafe sind in Wirklichkeit gesund! Der Test ist zwar "gut" (85% Sensitivität), aber weil das Virus so selten ist, überwiegen die falsch-positiven Ergebnisse massiv.

Visualisierung: 500 Schafe getestet
Gesund, Test negativ (473) Gesund, Test positiv (25) Infiziert, Test positiv (2) Infiziert, Test negativ (0–1)

Sofort behandeln — der Test ist ja positiv
Einen zweiten, unabhängigen Test durchführen
Den Test ignorieren — er ist zu ungenau
Alle Schafe sicherheitshalber behandeln

🧮 Schritt 4 — Der Satz von Bayes erklärt

Was wir gerade mit den 10'000 Schafen gerechnet haben, lässt sich als allgemeine Formel schreiben — den Satz von Bayes. Er beantwortet die Frage: Wie wahrscheinlich ist eine Ursache, wenn ich ein bestimmtes Ergebnis beobachte?
📐Die Bayes-Formel
                    p(B | A) × p(A)
  p(A | B)  =  ─────────────────────
                      p(B)
Was bedeuten die Symbole?
🎯p(A | B) = "Wie wahrscheinlich ist A, wenn B eingetreten ist?"
→ A = infiziert, B = Test positiv
→ p(infiziert | Test positiv) — das haben wir oben mit 6.4% berechnet
🔬p(B | A) = "Wie wahrscheinlich ist B, wenn A stimmt?"
→ B = Test positiv, A = infiziert
→ p(Test positiv | infiziert) = 0.85 — die Sensitivität des Tests (aus der Aufgabenstellung)
📊p(A) = "Wie häufig ist A insgesamt?"
→ A = infiziert
→ p(infiziert) = 1/250 = 0.004 — die Häufigkeit des Virus in der Herde (Prävalenz)
📋p(B) = "Wie häufig tritt B insgesamt auf?"
→ B = Test positiv
→ p(Test positiv) = 0.0532 — aus Aufgabe C (beide Pfade zu "positiv" addiert)
🧮Eingesetzt für unser Schafe-Beispiel
                     p(+|infiziert) × p(infiziert)
p(infiziert|+)  =  ───────────────────────────────
                            p(+)

                     0.85  ×  0.004
                =  ─────────────────
                       0.0532

                     0.0034
                =  ─────────
                    0.0532

                ≈  0.064  =  6.4%
💡Der Zähler (0.0034) ist der Pfad "infiziert UND positiv" aus dem Baumdiagramm — genau Aufgabe A!
💡Der Nenner (0.0532) ist die Gesamtwahrscheinlichkeit für "positiv" — genau Aufgabe C!
🔑Bayes sagt also: Von allen positiven Ergebnissen, welcher Anteil kommt von den wirklich Infizierten?
🏥Warum ist das wichtig im Alltag?
Dieses Phänomen betrifft nicht nur Schafe — es ist ein grundlegendes Problem aller medizinischen Tests.
1️⃣Seltene Krankheit + Test = viele falsche Alarme. Je seltener die Krankheit, desto mehr falsch-positive Ergebnisse. Bei 1/250 Häufigkeit und 5% Falsch-Positiv-Rate kommen auf jeden Kranken fast 15 Fehlalarme.
2️⃣Positiver Test ≠ sicher krank. Deshalb gibt es fast immer einen Bestätigungstest. Der erste Schnelltest ist nur eine Vorauswahl.
🔄Gegenbeispiel: Was wenn das Virus häufiger wäre?
Stell dir vor, 1 von 5 Schafen wäre infiziert (statt 1 von 250). Der Test bleibt genau gleich (85% Sensitivität, 5% Falsch-Positiv). Was ändert sich? 
Neues Baumdiagramm (1 von 5 infiziert):

Start
 ├─ Infiziert (1/5 = 0.20)          ← vorher: 1/250 = 0.004
 │    ├─ Test + (0.85)  → 0.20 × 0.85 = 0.170   ← vorher: 0.0034
 │    └─ Test − (0.15)  → 0.20 × 0.15 = 0.030   ← vorher: 0.0006
 └─ Gesund (4/5 = 0.80)             ← vorher: 249/250 = 0.996
      ├─ Test + (0.05)  → 0.80 × 0.05 = 0.040   ← vorher: 0.0498
      └─ Test − (0.95)  → 0.80 × 0.95 = 0.760   ← vorher: 0.9462

p(positiv) = 0.170 + 0.040 = 0.210            ← vorher: 0.0532
Bayes einsetzen:

                     0.85  ×  0.20      0.170
p(infiziert|+)  =  ──────────────── = ─────── ≈ 0.81 = 81%   ← vorher: 6.4%
                       0.210            0.210
📊Virus selten (1/250): Positiver Test → nur 6.4% Chance wirklich krank
📊Virus häufig (1/5): Positiver Test → ganze 81% Chance wirklich krank
🔑Gleicher Test, komplett anderes Ergebnis! Der entscheidende Faktor ist, wie häufig die Krankheit in der Herde vorkommt — die sogenannte Prävalenz.

Das Geburtstagsproblem

Warum 23 Personen reichen für eine 50%-Chance

🤔 Die überraschende Frage

Stell dir vor, du bist in einem Raum mit anderen Personen. Wie viele Personen müssen im Raum sein, damit die Wahrscheinlichkeit, dass mindestens zwei am selben Tag Geburtstag haben, grösser als 50% ist?
Was schätzt du?
Etwa 183 (die Hälfte von 365)
Etwa 50
Nur 23!
Etwa 100

🌳 Die Idee: Gegenereignis nutzen

Direkt zu rechnen, wie viele Geburtstags-Paare es gibt, wäre extrem kompliziert. Deshalb nutzen wir einen Trick: Wir berechnen das Gegenereignis — dass alle Geburtstage verschieden sind — und ziehen es von 1 ab.
1️⃣Person 1 betritt den Raum
Die erste Person kann an jedem der 365 Tage Geburtstag haben. Es gibt noch niemanden zum Vergleichen — die Wahrscheinlichkeit für "keinen Treffer" ist also: 
Person 1: 365/365 = 1  (100% — logisch!)
2️⃣Person 2 kommt dazu
Jetzt gibt es 1 "besetzten" Tag. Person 2 darf an 364 von 365 Tagen Geburtstag haben, um kein Match zu erzeugen: 
Person 1
 └─ neu (365/365)
     Person 2
      ├─ neu (364/365) — anderer Tag → kein Match
      └─ Treffer! (1/365) — gleicher Tag!
P(beide verschieden) = 365/365 × 364/365 = 0.99726
3️⃣Person 3 kommt dazu
Jetzt sind schon 2 Tage "besetzt". Person 3 hat nur noch 363 von 365 freie Tage: 
Person 1
 └─ neu (365/365)
     Person 2
      ├─ neu (364/365)
      │   Person 3
      │    ├─ neu (363/365) — noch ein anderer Tag
      │    └─ Treffer! (2/365)
      └─ Treffer!
P(alle 3 verschieden) = 365/365 × 364/365 × 363/365 = 0.99180
🔑Das Muster
Siehst du das Muster? Jede neue Person hat einen Tag weniger zur Auswahl. Bei Person n gibt es noch (365 − n + 1) freie Tage. Der Baum wächst immer weiter: 
Person 1:  365/365   (365 freie Tage)
Person 2:  364/365   (364 freie Tage)
Person 3:  363/365   (363 freie Tage)
Person 4:  362/365   (362 freie Tage)
   ...        ...
Person 23: 343/365   (343 freie Tage)
✖️Pfadregel: Entlang des "neu"-Pfades alle Faktoren multiplizieren
📐P(alle verschieden) = 365/365 × 364/365 × 363/365 × … × 343/365
💡Jeder Faktor ist etwas kleiner als 1 → das Produkt schrumpft immer schneller!
🔄Gegenereignis: Der letzte Schritt
Wir wollen aber P(mindestens ein Match) — das ist das Gegenereignis: 
P(mind. ein Match) = 1 − P(alle verschieden)

Bei 23 Personen:
  P(alle verschieden) = 365/365 × 364/365 × … × 343/365
                      ≈ 0.4927

  P(mind. ein Match)  = 1 − 0.4927
                      = 0.5073 = 50.73%
🎯Bei nur 23 Personen ist die Chance auf einen gemeinsamen Geburtstag bereits grösser als 50%!
🤔Warum so wenig? Weil es nicht um DEINEN Geburtstag geht — es geht um IRGENDEIN Paar. Bei 23 Personen gibt es 23 × 22 / 2 = 253 verschiedene Paare zum Vergleichen!

📝 Verständnisfragen

Warum rechnet man mit dem Gegenereignis "alle verschieden" statt direkt "mindestens ein Match"?
Weil das Gegenereignis immer einfacher ist
Weil "mindestens ein Match" viele verschiedene Fälle umfasst (1 Paar, 2 Paare, Drillinge...) — das Gegenereignis hat nur einen einzigen Pfad
Weil man mit dem Gegenereignis grössere Zahlen bekommt

⚠️ Aufgabe B — Wer ist überhaupt betroffen?

Der entscheidende Punkt: Nicht alle Risiken betreffen alle Menschen gleich! Manche Gefahren treffen jeden, andere nur die, die eine bestimmte Aktivität ausüben. Das macht den Vergleich in einer einzigen Tabelle problematisch.
🌍Kategorie 1: Betrifft (fast) alle
Diese Risiken treffen die gesamte Bevölkerung — man kann ihnen kaum ausweichen. Die Zahl in der Tabelle ist direkt aussagekräftig.
☄️Meteorit (0.0006) — trifft alle auf der Erde. Aber extrem unsicher, weil rein theoretisch modelliert (es gibt fast keine historischen Fälle).
🌍Erdbeben (17) — betrifft alle in Erdbebengebieten. In der Schweiz geringer als in Japan. Die Zahl ist ein weltweiter Durchschnitt.
🌊Überschwemmungen (22) — betrifft vor allem Bewohner von Flussgebieten und Küsten, aber grundsätzlich ein allgemeines Risiko.
🌪️Tornados (22) — stark regional abhängig (USA: Tornado Alley). In Europa viel seltener.
⚡Blitzschlag (3) — kann jeden treffen, der draussen ist. Die Zahl passt gut zu realen Daten (~0.3 pro Million).
✈️Herabfallendes Flugzeug (1) — betrifft alle, ist aber extrem selten.
🐍Giftige Tiere (2) — stark regional (in der Schweiz praktisch null, in Australien deutlich höher).
🤒Influenza (2'000) — betrifft potenziell alle, besonders ältere und immungeschwächte Menschen. Die Zahl ist plausibel.
🚗Von Fahrzeug überfahren (500) — betrifft alle Fussgänger und Verkehrsteilnehmer, also fast alle.
🎯Kategorie 2: Nur wenn man es tut!
Diese Risiken betreffen nur Personen, die die Aktivität ausüben. Wer nicht klettert, hat null Kletterrisiko! Trotzdem sind die Zahlen in der Tabelle auf die gesamte Bevölkerung bezogen — das "verdünnt" das Risiko massiv.
🧗Klettern (400) — nur Kletterer sind betroffen.

📝Zusammenfassung: 3 Lehren
1️⃣Risikotypen unterscheiden. Wer nicht klettert, hat null Kletterrisiko. Die Tabelle tut so, als wäre das Risiko für alle gleich — das ist irreführend.
2️⃣Regionale Unterschiede. Tornados sind in der Schweiz kein Thema, giftige Tiere auch nicht. Erdbeben treffen Japan stärker als Europa. Ein Weltdurchschnitt sagt wenig über dein persönliches Risiko.
3️⃣Datenqualität variiert enorm. Raucher-Daten basieren auf Langzeitstudien mit Millionen Teilnehmenden. Meteoriten-Risiken sind rein theoretisch. Motorrennen-Zahlen scheinen fragwürdig. Alles in einer Tabelle suggeriert eine Genauigkeit, die nicht existiert.
